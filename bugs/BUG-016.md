# BUG-016: Saving .aidignore freezes/crashes the sidebar nvim

**Status**: closed — fixed 2026-03  
**Branch**: main (`nvim/lua/sync.lua`, `nvim/lua/aidignore.lua`)  
**First appeared**: 2026-03  
**Fixed**: 2026-03  

---

## Symptoms

- User saves `.aidignore`.
- The treemux sidebar nvim freezes or loses the nvim-tree window.
- The sidebar pane becomes unresponsive; the tmux session must be killed and
  restarted to recover.
- `prefix+Tab` stops reopening the sidebar after the crash (separate root cause —
  see BUG-017).

## Root cause

Saving `.aidignore` triggered the following chain:

1. `aidignore.lua` `watch()` callback fires → `_cache = nil` → `_apply_to_nvimtree()`.
2. `_apply_to_nvimtree()` mutated `ignore_list`, called `api.tree.reload()`, then
   called `pcall(s.sync)` (i.e. `sync.sync()`).
3. `sync.sync()` called `_refresh_treemux_sidebar()`, which sent
   `require('aidignore').reset()` via msgpack-RPC to the sidebar nvim.
4. Inside the sidebar nvim, `reset()` called `_apply_to_nvimtree()` again →
   `pcall(s.sync)` again → `sync.sync()` → `vim.cmd("silent! checktime")`.
5. `checktime` detected that `.aidignore` had changed on disk and opened it as a
   new buffer in the sidebar nvim, replacing the nvim-tree window.

Two separate call sites both contributed:

| Call site | Problem |
|-----------|---------|
| `_apply_to_nvimtree()` in `aidignore.lua` | Called `pcall(s.sync)` after reloading the tree, which was unnecessary and triggered the RPC chain. |
| `_refresh_treemux_sidebar()` in `sync.lua` | Sent `require('aidignore').reset()` to the sidebar, which re-ran the full reset including `sync.sync()` and `checktime`. |

## Fix

**`nvim/lua/sync.lua`** — `_refresh_treemux_sidebar()`:  
Changed RPC payload from `require('aidignore').reset()` to
`pcall(require('nvim-tree.api').tree.reload)`.  
The sidebar only needs a tree refresh. It has its own `aidignore.watch()` running
directly for live filter updates; it does not need the editor nvim to push a reset.

**`nvim/lua/aidignore.lua`** — `_apply_to_nvimtree()`:  
Removed the `pcall(s.sync)` call. The sidebar tree is now refreshed exclusively by
the targeted RPC in `sync.lua`. Calling `sync.sync()` from inside
`_apply_to_nvimtree()` was incorrect in both the editor and sidebar contexts:
in the editor it was redundant (the editor's own watcher already handles sync),
and in the sidebar it triggered the fatal `checktime`.
