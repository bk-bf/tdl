# BUG-017: prefix+Tab does not reopen the sidebar after it is closed

**Status**: closed — fixed 2026-03-09  
**Branch**: main (`tmux.conf`)  
**First appeared**: 2026-03-09  
**Fixed**: 2026-03-09  

---

## Symptoms

- User presses `prefix+Tab` to reopen the treemux sidebar.
- Nothing happens (no new pane, no error).
- `ensure_treemux.sh` (called at session start) opens the sidebar correctly.
- The sidebar can be opened manually by running `toggle.sh` with the correct ARGS.

## Root cause

`tmux.conf` line 76:

```
bind Tab run-shell "#{E:AID_DIR}/tmux/plugins/treemux/scripts/toggle.sh '' ''"
```

`toggle.sh` expects:
- `$1` — ARGS: a 15-field comma-separated string encoding nvim command, init file,
  python path, position, size, editor position, editor size, focus mode, refresh
  intervals, debug flag, focus, and tree client.
- `$2` — PANE_ID: the tmux pane ID of the editor pane to attach the sidebar to.

Both arguments were passed as empty strings. With no ARGS, `toggle.sh` extracted
empty values for every field (position defaulted to empty, size to 0, etc.) and
could not open the sidebar. `PANE_ID` being empty meant `get_pane_info` returned
nothing, so `PANE_CURRENT_PATH` and `PANE_WIDTH` were also empty, preventing
`split_sidebar_left/right` from running.

The correct ARGS value is stored in the tmux server option `@treemux-key-Tab`,
written by `sidebar.tmux` (part of the treemux TPM plugin) at session start when
it processes the `@treemux-*` options set in `tmux.conf`.

`ensure_treemux.sh` worked because it explicitly reads `@treemux-key-Tab`:

```bash
ARGS=$(tmux -L aid show-option -gqv '@treemux-key-Tab')
```

The `bind Tab` line was not updated to match.

## Fix

**`tmux.conf`** — `bind Tab`:

```
# before
bind Tab run-shell "#{E:AID_DIR}/tmux/plugins/treemux/scripts/toggle.sh '' ''"

# after
bind Tab run-shell "#{E:AID_DIR}/tmux/plugins/treemux/scripts/toggle.sh '#{E:@treemux-key-Tab}' '#{pane_id}'"
```

`#{E:@treemux-key-Tab}` expands the option at runtime (same mechanism as
`#{E:AID_DIR}`). `#{pane_id}` passes the current pane's ID so `toggle.sh` can
look up the current path and width.
