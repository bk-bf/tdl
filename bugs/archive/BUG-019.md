# BUG-019: .aidignore plain names match as substrings, hiding files whose names contain the pattern

**Status**: closed — fixed 2026-03-09  
**Branch**: main (`nvim/lua/aidignore.lua`, `nvim-treemux/treemux_init.lua`)  
**First appeared**: 2026-03-09  

---

## Symptoms

- A file is present in the project and not listed in `.aidignore`, yet it is invisible in
  the nvim-tree sidebar and in Telescope.
- Other files created in the same directory with different names appear immediately.
- The file becomes visible after being renamed to something that does not contain any
  `.aidignore` entry as a substring.

**Observed example**: `T-003-T-022-test-environment.md` was hidden because `.aidignore`
contains `env` (intended to filter Python virtual-environment directories named `env`).
The filename contains the substring `env` inside `environment`, which triggered the filter.

## Root cause

nvim-tree's `filters.custom` list is matched via `vim.fn.match(path, pattern)` and
`vim.fn.match(basename, pattern)` (see `nvim-tree/explorer/filters.lua:182`). vimscript
`match()` is a **substring search** — a pattern `env` matches any string containing `env`
anywhere, not just a path component named exactly `env`.

`aidignore.lua:_build()` was inserting plain strings like `env` directly into
`pats.raw`, which were then fed verbatim to `filters.custom` (and to `ignore_list` on live
reload). The same raw strings were also fed by the startup block in `treemux_init.lua`
before `aidignore.watch()` took over.

As a result, every file or directory whose **name contains** any `.aidignore` entry as a
substring was silently hidden — even though the intent was to filter only path components
with that exact name.

## Fix

Anchor each plain name as a full path-component vimscript regex:

```
\(^\|/\)env\(/\|$\)
```

This matches `env` only when it is preceded by start-of-string or `/`, and followed by
`/` or end-of-string — i.e. only when `env` is a complete directory or file name, never
when it is a substring of a longer name like `environment.md`.

**`aidignore.lua:_build()`** — changed `table.insert(plain, p)` to:
```lua
table.insert(plain, [[\(^\|/\)]] .. p .. [[\(/\|$\)]])
```

**`treemux_init.lua` startup block** — the `filters.custom` initial population now applies
the same anchoring, and also skips entries containing `*` or `?` (glob characters not safe
in vimscript regex).

## Verification

```vim
let anchored = '\(^\|/\)env\(/\|$\)'
echo match('env', anchored)                          " 0  — exact name: filtered ✓
echo match('T-003-T-022-test-environment.md', anchored)  " -1 — substring: not filtered ✓
echo match('docs/env/foo', anchored)                 " 4  — path component: filtered ✓
echo match('docs/env', anchored)                     " 4  — trailing component: filtered ✓
```
